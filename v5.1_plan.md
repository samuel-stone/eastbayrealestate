```text

    |
```

# Completed Work

## 1. Autonomous AI Code Review

The system now performs automated reviews after commits.

Example:

```text
AI CODE REVIEW

Severity: Medium
Confidence: 80%

File:
scripts/run_post_commit_reports.py

Problem:
Potential database risk

Recommendation:
Improve validation and error handling
```

The current issue:

The model produces some false positives because it lacks full repository context.

Example:

- DATABASE_URL was flagged as a security issue
- Actual implementation uses environment secrets and parameterized SQL

This suggests the need for better context injection.

# 2. Agent Memory JSONB Upgrade

The original design stored JSON objects as TEXT.

The table was migrated:

```sql
ALTER TABLE agent_memory
ALTER COLUMN observation TYPE jsonb
USING observation::jsonb;
```

Added indexing:

```sql
CREATE INDEX idx_agent_memory_type
ON agent_memory ((observation->>'type'));
```

Current memory examples:

```json
{
  "type": "code_review",
  "commit": "8e72b1f",
  "files": [
    "scraper/scrape_redfin.py"
  ],
  "review": "..."
}
```

# Gemini + Ollama Strategy Questions

Potential approaches:

## Option A: Ollama First

```text
Request
 |
 v
Ollama
 |
 failure?
 |
 v
Gemini
```

Advantages:

- cheap
- private
- local
- predictable

Disadvantages:

- weaker reasoning
- slower on complex tasks

---

## Option B: Gemini First

```text
Request
 |
 v
Gemini
 |
 failure?
 |
 v
Ollama
```

Advantages:

- stronger reasoning
- better architecture reviews

Disadvantages:

- API cost
- external dependency

---

## Option C: Hybrid Routing

Example:

Simple tasks:

```text
formatting
lint review
basic bug checks

        |
        v

Ollama
```

Complex tasks:

```text
architecture decisions
security reviews
database design

        |
        v

Gemini
```
